Retrieval-Augmented Generation, commonly known as RAG, is a technique that combines
information retrieval with large language model generation. Instead of relying only
on what a language model learned during training, RAG systems first search a
knowledge base for relevant information and then pass that information to the model
as context before it generates an answer. This helps reduce hallucination and allows
the model to answer questions about documents it has never seen before.

A typical RAG pipeline has two major components: a Retriever and a Generator. The
Retriever is responsible for finding the most relevant pieces of text from a large
collection of documents. It usually works by converting both the documents and the
user's query into numerical vectors called embeddings, and then measuring the
similarity between these vectors. Documents whose embeddings are closest to the
query embedding are considered most relevant.

FAISS, developed by Meta AI, is one of the most popular libraries for storing and
searching embeddings efficiently. It allows a system to search through millions of
vectors in milliseconds by using specialized indexing structures. In a typical setup,
raw text is split into smaller chunks, each chunk is converted into an embedding
using a model such as a Sentence Transformer, and the embeddings are stored inside a
FAISS index for fast similarity search later.

The Generator component takes over once relevant chunks have been retrieved. It
combines the user's original query with the retrieved text chunks inside a single
prompt, and sends that combined prompt to a large language model. The model then
produces an answer that is grounded in the retrieved content rather than relying
purely on its internal training data. This combination of retrieval plus generation
is what makes RAG systems useful for building document assistants, customer support
bots, and research tools that can accurately answer questions about specific files.

Prompt engineering plays an important role in RAG systems. A well-designed prompt
usually instructs the model to answer only using the provided context, and to say
that it does not know the answer if the context does not contain enough information.
This prevents the model from making up facts, which is one of the biggest advantages
RAG has over a plain language model with no external knowledge source.
